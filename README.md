## 1) Написать сортировку для массива числовых данных от 200 тысяч элементов

**Пример:**

```
[15, 23, 1, -234, 400, …, 92]
```

**Примечание:**

Желательно получить результат максимально быстро, затрачивая минимум памяти при выполнении операций

**Итоговый результат:**

Получить отсортированный массив данных от меньшего к большему. Обосновать свое решение

### Решение

Код: `test1/sort.php`

```bash
php test1/sort.php          # 200 000 элементов
php test1/sort.php 500000
```

В продакшене использовал бы `sort($data, SORT_NUMERIC)` — это нативная реализация на C, сортирует in-place за O(n log n) и на 200k элементов примерно в 2 раза быстрее любого кода на PHP.

Чтобы показать, что понимаю алгоритмы, в том же файле написал итеративный QuickSort: median-of-three, insertion sort для малых диапазонов, явный стек вместо рекурсии. In-place, дополнительная память только на стек границ.

Другие варианты здесь не рассматривал: merge sort требует O(n) памяти, counting/radix — только при узком диапазоне значений, пузырёк и простая вставка — O(n²), на 200k неприемлемо.

## 2) Сформировать таблицы базы данных по условиям:

Необходимо разработать структуру базы данных для хранения игроков по сезонам.

**Клуб игрока должен содержать информацию:**

• Название клуба на русском  
• Название клуба на английском  
• Город клуба на русском  
• Город клуба на английском

**Информация об игроке:**

• ФИО на русском  
• ФИО на английском  
• Вес  
• Рост  
• Игровой номер игрока

Реализация свободная.

**Итоговый результат:**

Получить выгрузку из БД (можно с минимальным набором данных: 3 клуба, 2 сезона, 5 игроков). Выгрузка должна содержать структуру таблиц и набор данных (dump БД).

### Решение

Дамп: `test2/dump.sql`

```bash
mysql < test2/dump.sql
```

Структура из 4 таблиц: `clubs`, `seasons`, `players`, `player_seasons`. Клубы и сезоны — справочники. Постоянные данные игрока (ФИО, вес, рост) лежат в `players`. Связь «игрок — клуб — сезон» и игровой номер — в `player_seasons`: один игрок может выступать в разных клубах и под разными номерами в разные сезоны без дублирования ФИО.

Ограничения: уникальность номера в рамках клуба и сезона (`club_id + season_id + jersey_number`), уникальность записи игрока в клубе за сезон (`player_id + club_id + season_id`), внешние ключи с `ON DELETE CASCADE`.

В дампе — 3 клуба (ЦСКА, СКА, Локомотив), 2 сезона (2023/2024, 2024/2025), 5 игроков и 10 записей в составе: часть игроков меняет клуб или номер между сезонами.

Альтернатива с одной таблицей `players`, куда сразу кладутся `club_id`, `season_id` и номер, проще, но при повторном появлении игрока в новом сезоне дублируются ФИО, вес и рост. Для хранения «по сезонам» нормализованный вариант предпочтительнее.

## 3) Необходимо реализовать компонент выгрузки пользователей в формате CSV.

**Дано:**

• В базе данных более 500 000 пользователей  
• Необходимо по нажатию кнопки «Выгрузить пользователей» (можно реализовать без страницы в админ-панели)

**Примечание:**

• Страница пользователя не должна перезагружаться  
• Выгрузка должна выполняться асинхронно через AJAX-запрос  
• После завершения процесса должна быть предоставлена ссылка на скачивание файла либо автоматически открываться вкладка с загрузкой документа

**Поля пользователей для выгрузки:**

• Фамилия  
• Имя  
• Телефон  
• E-mail

**Итоговый результат:**

Получить CSV-файл с пользователями по нажатию кнопки

### Решение

Дамп: `test3/dump.sql` (500 000 пользователей, ~39 МБ)  
UI: `test3/index.php`  
Демо: [https://m-teams.ru/fhr_test3/](https://m-teams.ru/fhr_test3/)

По кнопке «Выгрузить пользователей» уходит AJAX `POST` на `api/start.php`. Сервер сразу отвечает JSON с `job_id`, затем через `fastcgi_finish_request()` в фоне пишет CSV на диск: небуферизованный `SELECT` по `users`, `fputcsv()` чанками по 10 000 строк с обновлением прогресса. Клиент опрашивает `api/status.php` каждые 1,5 с; по готовности — ссылка на скачивание и автоматический переход на `api/download.php` (доступ по токену).

Память не растёт с числом строк: данные идут потоком из БД в файл, без загрузки всего массива в PHP. Для 500k на DDEV выгрузка занимает около секунды.
